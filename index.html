1. What is an IDE and Why Do We Use It?

An IDE (Integrated Development Environment) is a software application that combines tools for writing, editing, debugging, and running code in one place. IDEs make the coding process more organized, efficient, and accessible, especially for beginners.

Why Use an IDE?

  • Convenience: IDEs provide features like autocompletion, syntax highlighting, and error correction, helping you write code faster and more accurately.
  • Debugging: Built-in tools for finding and fixing errors make testing easier.
  • Version Control: Many IDEs support integration with version control systems like Git, enabling you to track changes in your code.

Types of IDEs

There are many IDEs available, each suitable for different languages and types of projects. Here are some popular options:

  • PyCharm — popular for Python development.
  • IntelliJ IDEA — widely used for Java and other languages.
  • Eclipse — cross-platform IDE for Java and other languages.
  • Visual Studio Code — lightweight, multi-language editor that can function as a full IDE with extensions.

2. Visual Studio Code: Our IDE

Visual Studio Code is a free, open-source code editor developed by Microsoft. It’s suitable for writing code in various languages, including HTML, CSS, JavaScript, Python, and others. With extensions, VS Code can transform into a full IDE, making it ideal for both beginners and professionals.

Why Visual Studio Code?

  • Supports Multiple Languages: VS Code recognizes and highlights the syntax for many programming languages and can be extended to support more.
  • Extensions: VS Code has an extension marketplace where you can add new features for specific languages, themes, and tools.
  • Git Integration: Built-in Git support makes it easy to work with code repositories directly in the editor.
  • Lightweight and Flexible: VS Code is fast, doesn’t require much space, and works on most operating systems.

Example Workflow in Visual Studio Code

  • Installing Extensions — for example, for HTML and CSS, to enable autocompletion and styling hints.
  • Using the Terminal — VS Code includes a terminal for running commands directly in the editor, like package installations or project management.
  • File Management — a file navigation pane helps you manage and organize project files easily.

3. What is HTML?

HTML (HyperText Markup Language) is a markup language used to create the structure of web pages. It allows the browser to understand what elements should be displayed, in what order, and with what styling.

Key Points:

  • HTML defines headers, text, images, links, and other elements.
  • HTML is not a programming language; it is used solely to structure and present content.

4. Basic Elements of an HTML Document

An HTML document consists of tags. Tags are special commands that tell the browser what type of content or style should be applied to the content.

Essential Concepts:

  • Tag: The fundamental unit in HTML, written within angle brackets (<>).
  • Element: A tag with its content (for example, <h1>Heading</h1>).
  • Attribute: Additional parameters that specify how an element should work or display (for example, <img src="image.jpg" alt="Image description">).

5. Structure of an HTML Document

An HTML document contains two main parts: <head> and <body>.

<!DOCTYPE html>
                <html>
                <head>
                    <meta charset="UTF-8">
                    <meta name="viewport" content="width=device-width, initial-scale=1.0">
                    <meta name="description" content="A basic example of an HTML page structure">
                    <title>My First Page</title>
                    <link rel="stylesheet" href="styles.css">
                    <link rel="icon" href="favicon.ico" type="image/x-icon">
                    <script src="script.js" defer></script>
                </head>
                <body>
                    <h1>Welcome to my page!</h1>
                    <p>This is the first paragraph on my web page.</p>
                </body>
                </html>

Structure Explanation:

  • <!DOCTYPE html>: Declares the document as HTML5.
  • <html>: The root tag containing the entire HTML document.
  • <head>: Contains essential information about the page, such as:
    • <title>: Defines the page title that appears in the browser tab.
    • <meta charset="UTF-8">: Sets the character encoding for the page, ensuring text displays correctly in the browser.
    • <meta name="viewport" content="width=device-width, initial-scale=1.0">: Controls the page’s viewport settings, helping the layout adjust for different screen sizes, which is important for mobile optimization.
    • <meta name="description" content="... ">: Provides a brief description of the page’s content, useful for SEO and search engines to display in search results.
    • <link rel="stylesheet" href="styles.css">: Links to an external CSS file that defines the page’s style and layout.
    • <link rel="icon" href="favicon.ico" type="image/x-icon">: Sets the favicon (the small icon that appears on the browser tab for the page).
    • <script src="script.js" defer></script>: Loads an external JavaScript file with the defer attribute, which tells the browser to execute the script after the HTML document has been parsed, improving page load performance.
  • <body>: The main content of the page, visible to the user (headings, text, images, etc.).

6. Basic HTML Tags

Examples of Common HTML Tags:

  • <h1> to <h6>: Used for creating headings of different levels.
  • <p>: Used for text paragraphs.
  • <a href="...">: Used to create links to other pages.
  • <img src="..." alt="...">: Used to display images on the page.

Example usage:

<h1>Heading Level 1</h1>
<p>This is a paragraph of text.</p>
<a href="https://example.com">Go to the website</a>
<img src="image.jpg" alt="Description of the image">

7. Attributes in HTML

Attributes in HTML are extra bits of information that provide additional details about elements. They modify the behavior or appearance of an element and are always included inside the opening tag. Attributes follow a name="value" structure, where the name defines the type of attribute and the value provides specific details.

<p class="myClass">This is a paragraph of text.</p>

Commonly Used Attributes:

  • id — A unique identifier for an element on the page.
    • Often used for styling specific elements with CSS or for targeting elements with JavaScript.
    <div id="unique-box">This is a uniquely identified box.</div>
  • class — Defines one or more classes for an element, allowing multiple elements to share the same styling or behavior.
    • Can be used for styling groups of elements with CSS.
    <p class="highlight">This paragraph has a special class.</p>
  • style — Adds inline CSS styling directly to an element.
    • Useful for quick styling changes, although it’s better to use CSS stylesheets for larger projects.
    <div style="background-color: lightblue; padding: 10px;">Styled div with background color.</div>
    Styled div with background color.
  • href — Used in <a> tags to define the URL of a link.
    • Specifies the destination of a hyperlink.
    <a href="https://example.com">Visit Example</a>
  • src — Specifies the source URL for an image in an <img> tag.
    • Provides the path to the image file.
    <img src="image.jpg" alt="Description of image">
  • alt — An alternative text for an image.
    • Essential for accessibility, providing a text description for images.
    <img src="image.jpg" alt="Description of the image">
  • target — Used in links (<a>) to specify where to open the linked document.
    • Common values include _blank (opens in a new tab) and _self (opens in the same tab).
    <a href="https://example.com" target="_blank">Open in new tab</a>

8. Using Semantic HTML for Accessibility

In HTML, you can place all content inside <div> tags, but this approach has major limitations for accessibility and SEO. Semantic tags like <h1>, <p>, <article>, <header>, and others provide a clear structure that’s beneficial for both search engines and screen readers, which help visually impaired users.

Why Semantic Tags Matter:

  • Improved SEO: Search engines understand the purpose of each tag, which helps with ranking and visibility.
  • Accessibility: Screen readers, which visually impaired users rely on, use these tags to convey content structure. Tags like <header>, <nav>, <main>, <footer>, and <article> help users navigate the content more easily.

Example of Semantic HTML:

<article>
    <header>
        <h1>Accessible Web Design</h1>
    </header>
    <p>Creating content with semantic tags helps everyone navigate and understand the page better.</p>
</article>

Example of Non-Semantic HTML (using only <div> tags):

<div>
    <div>
        <h1>Accessible Web Design</h1>
    </div>
    <div>Creating content with semantic tags helps everyone navigate and understand the page better.</div>
</div>

9. Simple HTML Page Example

Let’s put together what we’ve learned to create a basic HTML page with a title, text, a link, and an image.

<!DOCTYPE html>
<html>
<head>
    <title>My First HTML Page</title>
</head>
<body>
    <h1>Hello, HTML!</h1>
    <p>This is my first HTML document. I'm just starting to learn web development.</p>
    <a href="https://example.com" target="_blank">Visit the Example site</a>
    <br>
    <img src="image.jpg" alt="Beautiful picture">
</body>
</html>

Explanation:

  • <h1> is used for the page title.
  • <p> is for the paragraph text.
  • <a> creates a link to an external site.
  • <img> displays an image on the page.

10. How HTML Works

When you open an HTML document in the browser, the browser:

  1. Reads the HTML code and converts it into the DOM (Document Object Model).
  2. Displays each element on the page according to the specified HTML tags and attributes.
